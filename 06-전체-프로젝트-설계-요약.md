## 6.2 핵심 설계 결정 요약

### 왜 네이버 DataLab인가?
한국 검색 시장 점유율 64%, 2007년부터 일별 데이터, 무료 API. Google Trends(26%, 샘플링 편향), 카카오(3.4%, 트렌드 API 없음), 트위터(과거 데이터 없음) 대비 압도적 우위. → [상세: 02-데이터-소스-조사-및-선정.md](./02-데이터-소스-조사-및-선정.md)

### 왜 썸트렌드를 추가하는가?
검색량만으로는 "왜 뜨는지"를 모름. 소셜 언급량과 감성 데이터가 검색량보다 **3~7일 선행**하는 패턴이 학술적으로 확인됨. 교차 피처(mention_to_search_ratio, lag_correlation)가 모델의 예측력을 높임. → [상세: 03-피처-엔지니어링-설계.md](./03-피처-엔지니어링-설계.md)

### 왜 virality_score (연속값)인가?
이진 분류(바이럴/비바이럴)로 하면 클래스 불균형(5% vs 95%)이 심각. 연속값으로 바꾸면 모든 키워드가 고유 타겟을 가져 불균형 문제 소멸. 정보량도 증가("얼마나 바이럴인가" 구분 가능). LSTM의 데이터 부족 문제도 해결(40개 → 42만 개). → [상세: 01-프로젝트-개요-및-방향-결정.md](./01-프로젝트-개요-및-방향-결정.md)

### 왜 XGBoost + LSTM 병렬 구조인가?
XGBoost: 직접 설계한 파생 피처 기반, 설명 가능(SHAP), 빠른 학습(초 단위), 표 데이터에서 SOTA 수준.
LSTM: 원본 시계열 자동 패턴 추출, 피처 엔지니어링 없이도 작동, "우리가 생각 못한 패턴" 포착 가능.
"전통 ML vs 딥러닝 비교"가 캡스톤의 핵심 학술 포인트. → [상세: 04-모델-선택-및-비교-분석.md](./04-모델-선택-및-비교-분석.md)

### 왜 2022년부터인가?
COVID-19(2020~2021)가 구조적 단절을 일으켜 pre-COVID 데이터가 현재 패턴과 불일치. 4년(2022~현재)이면 바이럴 사례 40~80개로 EPP ≥ 2.0 충족, 트렌드 사이클(6년) 대부분 커버. → [상세: 02-데이터-소스-조사-및-선정.md §2.7](./02-데이터-소스-조사-및-선정.md)

### 왜 핵심 연관어 빈도와 경제지표를 피처에 추가하는가?
검색량/언급량 같은 행동 데이터만으로는 "동일한 상승 추세에서 어떤 키워드가 더 바이럴 될까"를 구분 못함. 썸트렌드 연관어를 TF-IDF로 벡터화하고 PCA로 20차원 축소하면, 키워드의 의미적 특성(고급/수입/편의점 등)을 모델이 자동으로 학습. 경제지표(CPI, CSI)는 시기별 소비 환경 context를 제공. 사전 만들기나 수동 태깅 없이 sklearn 5줄로 완전 자동화. → [상세: 03-피처-엔지니어링-설계.md §3.6~3.7](./03-피처-엔지니어링-설계.md)

### 왜 컴퓨팅 비용이 거의 0원인가?
XGBoost는 노트북에서 수 분, LSTM은 Colab 무료 T4 GPU에서 12분 내 학습 완료. 40~80만 샘플 규모에서는 유료 클라우드가 불필요. Colab 무료 + Kaggle 무료 + 학생 크레딧($300~2,000)으로 프로젝트 전체를 커버. → [상세: 04-모델-선택-및-비교-분석.md §4.7](./04-모델-선택-및-비교-분석.md)

### 왜 블로그/뉴스 채널만 사용하는가?
학습(프리미엄)과 추론(무료 티어)의 데이터 일관성 유지(train-serving skew 방지). 무료 티어가 블로그/뉴스만 제공하므로, 학습도 동일 채널만 사용. 발표 데모 시 유료 의존 없이 작동하는 시스템 완성. → [상세: 02-데이터-소스-조사-및-선정.md §2.5](./02-데이터-소스-조사-및-선정.md)

## 6.3 예상 결과물

### 1단계 (메인): 바이럴 강도 예측

```
입력: 키워드 ("약과")
출력:
  XGBoost: virality_score = 4.2  →  "바이럴 가능성 높음"
  LSTM:    virality_score = 3.8  →  "바이럴 가능성 높음"

  SHAP 설명:
    search_velocity_7d  = +2.3 기여
    positive_ratio      = +1.1 기여
    mention_velocity_7d = +0.8 기여
```

### 2단계 (보너스, 시간 여유 시): 검색 트렌드 곡선 예측

```
입력: 키워드 + 과거 60일 시계열
출력: 향후 30일 일별 검색 트렌드 예측 곡선
  → 피크 도달 예상 날짜 자동 추출
```

2단계는 시계열 예측 모델(LSTM Forecaster 또는 Prophet) 별도 구현 필요. 1단계가 확실히 완성된 후 추가.
