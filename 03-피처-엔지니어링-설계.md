# 03. 피처 엔지니어링 설계

## 3.1 피처 설계 원칙

모델에 입력되는 피처(feature)는 "이 키워드가 바이럴 될 조짐이 있는가?"를 판단할 수 있는 수치 정보여야 함. 키워드 이름(문자열) 자체는 피처가 아니며, 해당 키워드에 대한 **정량적 측정값**이 피처.

피처는 역할별로 4개 그룹으로 나뉨:
- **A. 검색 모멘텀**: 검색 트렌드의 속도와 방향
- **B. 소셜 버즈**: SNS/블로그/뉴스에서의 반응
- **C. 교차 상관**: 검색과 소셜 간의 관계
- **D. 외부 컨텍스트**: 시간·계절 정보

총 피처 수: **약 15~20개**. 학술 연구에서 바이럴 예측 모델은 보통 15~40개 피처 사용. 학부 캡스톤에서는 20개 내외가 관리 가능한 범위.

---

## 3.2 Group A: 검색 모멘텀 피처 (네이버 DataLab 기반)

"이 키워드가 지금 얼마나 빠르게 뜨고 있는가"를 수치화.

### A1. search_velocity_7d — 7일 검색 변화율

```
search_velocity_7d = (trend_today - trend_7days_ago) / (trend_7days_ago + 1)
```

- **의미**: 최근 7일간 검색 트렌드의 상승/하락 속도
- **예시**: "약과"가 7일 전 트렌드값 20, 오늘 45 → velocity = (45-20)/(20+1) = 1.19 (119% 상승)
- **분모 +1**: 7일 전 값이 0일 때 분모가 0이 되는 것 방지

### A2. search_acceleration — 검색 가속도

```
search_acceleration = velocity_today - velocity_yesterday
```

- **의미**: velocity 자체의 변화율. 양수면 "갈수록 빨라지는 상승", 음수면 "상승 둔화"
- **예시**: 어제 velocity 0.5, 오늘 velocity 1.19 → acceleration = 0.69 (가속 중)
- **핵심**: 바이럴 직전에는 acceleration이 급등하는 패턴이 관찰됨

### A3. ma_ratio — 단기/장기 이동평균 비율

```
ma_7 = 최근 7일 검색 트렌드 평균
ma_30 = 최근 30일 검색 트렌드 평균
ma_ratio = ma_7 / (ma_30 + 1)
```

- **의미**: 주식 분석의 골든크로스/데드크로스와 동일 원리. ma_ratio > 1이면 "단기 추세가 장기 추세를 상회" = 최근 상승세
- **예시**: ma_7 = 45, ma_30 = 22 → ma_ratio = 2.05 (단기가 장기의 2배 = 급격한 상승)

### A4. search_volatility_7d — 검색 변동성

```
search_volatility_7d = 최근 7일 일별 트렌드값의 표준편차
```

- **의미**: 높으면 불안정한 급등/급락 (일시적 이슈), 낮으면 안정적 추세
- **용도**: 같은 velocity라도 volatility가 높으면 "일시적 스파이크", 낮으면 "지속적 상승"

### A5. trend_slope_30d — 30일 선형 추세 기울기

```
30일간 (일자, 트렌드값) 데이터에 선형회귀 적합 → 기울기(slope) 추출
```

- **의미**: 최근 30일간의 전반적 상승/하락 방향과 강도
- **velocity와의 차이**: velocity는 "오늘 vs 7일 전" 두 점만 비교, slope는 30일 전체 추세

## 3.3 Group B: 소셜 버즈 피처 (썸트렌드 기반)

### B1. mention_volume_daily — 일별 총 언급량

```
mention_volume_daily = 블로그 언급량 + 뉴스 언급량
```

- **의미**: 해당 키워드가 오늘 블로그와 뉴스에서 총 몇 건 언급됐는지
- **주의**: 채널별로 기본 언급량 규모가 다르므로, 절대값보다 변화율이 더 유용

### B2. mention_velocity_7d — 언급량 7일 변화율

```
mention_velocity_7d = (mention_today - mention_7days_ago) / (mention_7days_ago + 1)
```

- **의미**: 검색 velocity와 동일 계산이지만 소셜 언급량 기반
- **핵심 관찰 (학술 근거)**: 다수의 정보 확산 연구(Information Cascade 모델)에서 **SNS 언급이 검색량보다 3~7일 먼저 움직이는** 선행 패턴이 확인됨. 소셜 미디어에서 콘텐츠가 먼저 확산되고, 이를 본 사용자들이 검색 엔진에서 해당 키워드를 검색하는 행동 흐름. 즉 mention_velocity가 먼저 치솟고, search_velocity가 3~7일 후 따라 올라감. 이 시차(lag)가 바로 **예측의 기회 창(window)**이 됨 — SNS 급등을 감지하면 검색 급등을 미리 예측할 수 있다는 뜻

### B3. positive_ratio — 긍정 감성 비율

```
positive_ratio = 긍정 언급 건수 / (전체 언급 건수 + 1)
```

- **의미**: 전체 언급 중 긍정적 반응의 비율
- **예시**: 전체 200건 중 긍정 120건 → positive_ratio = 0.6

### B4. sentiment_momentum — 감성 변화율

```
sentiment_momentum = positive_ratio_today - positive_ratio_7days_ago
```

- **의미**: 긍정 비율이 올라가고 있는지 내려가고 있는지
- **왜 중요한가**: 논문에서 **절대 감성값보다 감성의 변화 방향이 바이럴 예측력이 더 높다**고 보고. "긍정이 많다"보다 "점점 긍정이 늘고 있다"가 더 강한 신호
- **예시**: 7일 전 positive_ratio 0.5 → 오늘 0.7 → momentum = +0.2 (긍정 급증)

### B5. blog_news_ratio — 블로그/뉴스 비율

```
blog_news_ratio = 블로그 언급량 / (뉴스 언급량 + 1)
```

- **의미**: 블로그(일반인 자발적 콘텐츠)와 뉴스(미디어 보도)의 비율
- **패턴**: 바이럴 초기에는 blog_news_ratio가 높고(일반인이 먼저 관심), 후기에는 뉴스가 따라붙으면서 비율이 낮아짐

## 3.4 Group C: 교차 상관 피처 (두 소스 결합)

이 그룹이 **가장 차별화된 피처**. 검색 데이터와 소셜 데이터의 관계에서 나오는 신호.

### C1. mention_to_search_ratio — 언급/검색 비율

```
mention_to_search_ratio = mention_volume_daily / (search_trend_today + 1)
```

- **의미**: "SNS에서는 많이 언급되는데 아직 검색은 안 하는" 상태를 포착
- **해석**:
  - 이 값이 높으면 → "아직 SNS에서만 뜨고 검색으로 안 넘어간 상태" = **앞으로 검색량 상승 여지**
  - 이 값이 낮으면 → "이미 검색까지 넘어온 상태" = 상승 여지 제한적

### C2. social_search_lag_correlation — 소셜→검색 시차 상관

```
social_search_lag_correlation = corr(mention[t-7:t], search_trend[t-4:t+3])
```

- **의미**: 최근 7일간의 언급량 패턴과 3~7일 뒤 검색 트렌드 패턴의 상관계수
- **핵심 원리**: SNS 언급 → 3~7일 후 검색량 상승이라는 시차 관계가 존재할 때, 이 상관계수가 높게 나옴
- **높으면**: "소셜 버즈가 검색으로 전이되는 중" = 바이럴 진행 중 신호

## 3.5 Group D: 외부 컨텍스트 피처

### D1. day_of_week — 요일 (0~6)

- 주말/주중 효과. 식품 검색은 주말에 패턴이 다를 수 있음

### D2. month — 월 (1~12)

- **계절성이 매우 강한 피처**. 빙수류 7~8월, 붕어빵/호떡 12~1월, 발렌타인데이 초콜릿 2월 등
- 과자/베이커리 카테고리에서 특히 중요

### D3. is_holiday — 공휴일 여부 (0 or 1)

- 명절(설, 추석), 크리스마스, 어린이날 등에 특정 식품 검색 급등

### D4. competing_trends_count — 동시 경쟁 트렌드 수

```
같은 시기에 과자/베이커리 카테고리에서 virality_score > 1.5인 다른 키워드 수
```

- **의미**: 경쟁이 많으면 개별 키워드의 바이럴 지속 시간이 짧아짐
- **예시**: 동시에 3개 식품이 뜨고 있으면 관심이 분산되어 각각의 피크가 낮아지는 경향

## 3.6 XGBoost vs LSTM 입력 형태 차이

**같은 데이터를 다른 형태로 가공해야 함.**

### XGBoost 입력: 1행 × 20열 (피처 스냅샷)

특정 (키워드, 날짜) 조합에 대해 위 피처들을 모두 계산한 **한 줄의 벡터**.

```
키워드: 약과, 날짜: 2024-03-15
[velocity_7d=1.19, acceleration=0.69, ma_ratio=2.05, volatility=3.2,
 slope_30d=0.8, mention_volume=180, mention_velocity=1.5,
 positive_ratio=0.7, sentiment_momentum=0.2, blog_news_ratio=3.5,
 mention_search_ratio=2.8, lag_corr=0.72,
 day_of_week=4, month=3, is_holiday=0, competing=2, ...]
→ virality_score = ?
```

### LSTM 입력: 60행 × 8열 (60일 시계열 행렬)

**파생 피처를 직접 계산하지 않고**, 원본 시계열을 넣어서 LSTM이 패턴을 자동 추출하게 함.

```
키워드: 약과, 기준일: 2024-03-15 (과거 60일)

     search_trend  blog_mention  news_mention  positive_ratio  ...
Day1      12           30            5            0.50
Day2      13           28            6            0.52
Day3      11           32            4            0.48
...
Day59     58           150           28           0.72
Day60     67           180           31           0.78

→ virality_score = ?
```

**LSTM 입력의 정규화 필수**: LSTM은 입력값 스케일에 민감. 각 피처를 0~1 범위로 Min-Max 정규화하거나 표준화(평균 0, 표준편차 1) 처리해야 함. XGBoost는 트리 기반이라 정규화 불필요.
