# 02. 데이터 소스 조사 및 선정

## 2.1 한국 시장 데이터 소스 전수 비교

바이럴 예측 모델에 사용할 수 있는 한국 시장 데이터 소스를 전수 조사하고 비교한 결과.

### 주요 소스 비교표

| 소스 | 한국 커버리지 | 과거 데이터 | 일별 단위 | 공개 API | 무료 | 데이터 품질 |
|------|-------------|------------|----------|---------|------|-----------|
| **네이버 DataLab** | ★★★ (점유율 64%) | 2007년\~ | O | O | O | ★★★ |
| **Google Trends (pytrends)** | ★☆☆ (점유율 26%) | 가변적 | △ (1\~7일만) | X (비공식) | O | ★☆☆ |
| **카카오/다음** | ★☆☆ (점유율 3.4%) | X | X | △ (검색결과만) | △ | ★☆☆ |
| **트위터/X** | ★☆☆ (한국 사용량 낮음) | X (실시간만) | X | O | △ | ★★☆ |
| **YouTube Data API** | ★★★ (4,400만 사용자) | △ | X | O | O | ★★☆ |

### 소스별 상세 분석

#### 네이버 DataLab (최종 선정 — 1순위)

- **엔드포인트**: `https://naveropenapi.apigw.ntruss.com/datalab/v1/search`
- **인증**: 네이버 클라우드 플랫폼 등록 → Client ID / Client Secret (HTTP 헤더)
- **데이터 형태**: 0\~100 정규화 상대값 (조회 기간 내 최고점 = 100)
- **시간 단위**: 일별(`date`), 주별(`week`), 월별(`month`)
- **과거 데이터**: 2007년 1월\~현재 (18년+)
- **요청 제한**: 한 번에 키워드 그룹 5개, 그룹당 키워드 20개
- **Rate Limit**: 하루 1,000건
- **비용**: 무료

**한계:**
- 0\~100 정규화 값이라 절대 검색량 불가. 조회 기간 변경 시 같은 키워드도 값이 달라짐 → **모든 키워드를 동일 기간으로 조회 필수**
- 실시간 검색 서비스 2021년 3월 폐지 → 과거 트렌드 분석 전용

**쇼핑인사이트 연동:**
- **엔드포인트**: `/v1/datalab/shopping/category/keywords`
- 식품 카테고리 ID: `50000008`
- **Top 500 인기검색어** 기간별 조회 가능
- 쇼핑인사이트에서 키워드 목록 확보 → 검색어트렌드 API로 일별 트렌드 수집하는 파이프라인 구축 가능

#### Google Trends (pytrends) — 한국 시장 부적합, 탈락

- 한국 내 구글 점유율 약 26%로 한국 키워드 데이터 빈약
- 공식 API 없음, 비공식 라이브러리(pytrends) 사용 시 IP 차단 빈번
- 일별 데이터는 1\~7일 범위에서만 가능
- **샘플링 편향 심각**: 같은 키워드를 다른 날 조회하면 값이 달라짐
- 결론: 한국 식품 바이럴 예측에는 부적합

#### 카카오/다음 — API 자체 없음, 탈락

- 검색 시장 점유율 3.4%로 매우 낮음
- 트렌드 분석 API 자체가 존재하지 않음 (검색 결과 API만 존재)
- 2026년 Upstage 이전 등 조직 불확실성
- 결론: 프로젝트 대상에서 완전 배제

#### 트위터/X — 과거 데이터 없음, 보조만 가능

- 실시간 트렌딩 리스트만 제공, **과거 데이터 아카이브 없음**
- 한국 내 트위터 사용률이 카카오톡/유튜브 대비 낮음
- 시계열 예측 모델 학습용으로 사용 불가
- 결론: 실시간 보조 신호로만 활용 가능하나, 본 프로젝트에서는 사용하지 않음

#### YouTube Data API — 영상 특화, 일반 키워드 부적합

- 한국 사용자 4,400만으로 높은 커버리지
- 트렌딩 비디오, 조회수 통계 등 제공
- **한계**: 영상/음악 트렌드에 한정, 일반 식품 검색 키워드 트렌드와 성격 다름
- 결론: 식품 바이럴 예측의 주 데이터 소스로 부적합

#### 썸트렌드 (Sometrend) — 2순위, 소셜 피처 보강용

## 2.3 데이터 소스 약관 검토

### 네이버 DataLab API 이용약관

> "고객은 회사의 사전 동의 없이 본 서비스의 결과 데이터를 본 약관에서 허용한 범위를 넘어서서 무단으로 **복제, 저장, 가공, 배포**하거나 제3자에게 제공해서는 안 됩니다."

**학술 목적 별도 허가 가능성 검토:**

네이버 개발자센터 제휴신청의 공식 **반려 사유**에 캡스톤 프로젝트가 직접 해당:

1. "API를 테스트 또는 학습의 목적으로 사용하는 경우" → **반려**
2. "API 결과 데이터의 추출 및 저장 목적" → **반려**
3. "API 용도가 실제 서비스와 다르거나 약관상 허용하지 않는 목적으로 사용하는 경우" → **반려**

**현실적 판단:**

- 네이버가 학부 캡스톤 프로젝트의 API 호출을 추적하여 법적 조치를 취할 가능성: 극히 낮음
- Rate limit(하루 1,000건) 내에서 조용히 수집하면 기술적으로 감지 불가
- 한국 대학 캡스톤 프로젝트의 대다수가 동일한 방식으로 공개 API 데이터를 수집 사용 중
- **진짜 문제가 되는 경우**: 수집 데이터를 외부 재배포, 상업적 사용, 논문 게재 시
- **학부 수업 발표에서 끝나면 실질적 리스크: 0에 가까움**

**발표 자료에서의 표기**: "네이버 DataLab 공개 API를 통해 수집"으로만 기술.

## 2.5 추론 시 데이터 접근 전략 (Train-Serving Skew 방지)

### 문제 인식

학습 시 사용한 피처를 추론 시에도 동일하게 제공해야 함. 썸트렌드 프리미엄을 해지하면 추론 시 데이터 접근이 제한됨.

### 해결: 학습과 추론 모두 블로그/뉴스 채널만 사용

| 단계 | 소스 | 채널 | 비용 |
|------|------|------|------|
| 학습 | 썸트렌드 프리미엄 (1개월) | 블로그 + 뉴스 (6채널 중 2개만 선택 사용) | 180,000원 |
| 추론/데모 | 썸트렌드 무료 티어 | 블로그 + 뉴스 (무료 범위) | 0원 |
| 추론 보조 | 네이버 DataLab API | 검색 트렌드 실시간 조회 | 0원 |

**핵심 설계 원칙**: 학습에 6개 채널 전부 쓰고 추론에 2개 채널만 쓰면 **train-serving skew** 발생 → 성능 하락. 따라서 학습 때부터 블로그/뉴스만 사용하여 일관성 유지.

**이 전략의 추가 이점:**
1. 발표 데모 시 유료 서비스 의존 없이 작동 → 완성도 면에서 인상적
2. 심사 시 "유료 서비스 끊으면 안 되는 거 아님?" 질문 방어 가능
3. 블로그/뉴스가 식품 트렌드의 주요 전파 채널 (인스타/유튜브 언급량은 engagement가 아닌 단순 언급 수라 추가 가치 제한적)

## 2.7 수집 기간 결정: 2022년 1월 \~ 현재 (약 4년)

### 근거 ①: COVID-19 구조적 단절

2020\~2021년 상반기 데이터는 소비자 행동이 비정상적. 학술 연구에서 2020년 초를 기점으로 1% 유의수준의 **구조적 변화점(structural break)** 확인. Pre-COVID와 post-COVID 데이터를 혼합하면 모델 성능 저하.

2022년부터는 post-COVID "뉴노멀" 패턴 안정화 확인.

> **출처**: "COVID Impact on Consumption Behaviors" — Scientific Reports (2024), Nature Communications

### 근거 ②: 희귀 이벤트 최소 샘플 수

바이럴 식품 키워드는 연간 약 10\~20건 발생. 학술 기준 EPP(Events Per Predictor) ≥ 23 필요.

| 기간 | 바이럴 사례 수 (추정) | EPP (피처 20개 기준) |
|------|----------------------|---------------------|
| 2년 | 20\~40개 | 1.0\~2.0 (**부족**) |
| 3년 | 30\~60개 | 1.5\~3.0 (아슬아슬) |
| **4년** | **40\~80개** | **2.0\~4.0 (적정)** |
| 5년+ | 50\~100개 | 2021 코로나 잔여 노이즈 포함 |

> **출처**: "Minimum Sample Size for Prediction Models" — PMC (2019)

### 근거 ③: 트렌드 사이클 가속화

식품 트렌드 채택 주기가 과거 12년 → 최근 6년으로 압축.

> **출처**: "Food Trend Forecasting with Demographics" — Nature (2024)

5년 전 데이터는 이미 "다른 시대"의 패턴. 4년이 현재 사이클 대부분 커버 + 노이즈 최소화 적정선.

### 최종 데이터 기간

```
수집: 2022.01.01 ~ 2026.03 (현재)
실사용 (피처 계산 가능 구간): 2022.04경 ~ 2025.08경
  - 처음 90일: 기준선(baseline) 피처 계산에 소요
  - 마지막 30일: 타겟(향후 30일 피크) 계산 불가
```
