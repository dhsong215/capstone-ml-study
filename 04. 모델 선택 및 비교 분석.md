| 모델 | 역할 | 입력 형태 | 출력 |

| **Dummy Model** | 베이스라인 ① (하한선) | 없음 (학습 데이터 평균) | virality_score + peak_timing |
| **Linear Regression** | 베이스라인 ② (선형) | 피처 스냅샷 43개 | virality_score + peak_timing |

| **LSTM Regressor** | 메인 모델 ② | 60일 시계열 + 연관어 벡터 20차원 | virality_score + peak_timing |

**Dual Target 구조**: virality_score(얼마나 급등)와 peak_timing(몇째 날에 피크)을 각각 별도 모델로 학습. 동일 피처·동일 아키텍처에서 타겟만 교체하므로 추가 비용은 학습 시간 × 2뿐 (XGBoost 30초, LSTM 24분 — 여전히 Colab 무료로 충분).

## 4.2 XGBoost — 원리와 동작 메커니즘

### Gradient Boosting의 단계별 동작

XGBoost는 **결정 트리(Decision Tree)**를 순차적으로 쌓아올리는 앙상블 기법. 각 트리가 **이전 트리들이 틀린 부분**을 교정하도록 학습.

#### Step 1: 초기 예측

전체 키워드에 대해 단순한 초기 예측. 예: virality_score의 전체 평균 = 1.2에서 시작.

#### Step 2: 잔차(Residual) 계산

```
"약과"  → 실제 score 6.2, 예측 1.2 → 잔차 = +5.0 (많이 틀림)
"초코파이" → 실제 score 0.8, 예측 1.2 → 잔차 = -0.4 (조금 틀림)
```

XGBoost는 잔차뿐 아니라 **2차 미분(Hessian)**도 계산. 이는 "loss가 얼마나 빠르게 변하는지"를 알려줘서 더 정밀한 보정이 가능.

#### Step 3: 잔차를 예측하는 결정 트리 생성

```
search_velocity_7d > 1.5?
  ├─ Yes → positive_ratio > 0.7?
  │         ├─ Yes → 잔차 예측: +4.2
  │         └─ No  → 잔차 예측: +1.8
  └─ No  → 잔차 예측: -0.1
```

트리가 "velocity가 높고 감성이 긍정적이면 큰 양의 잔차(= 바이럴 방향 보정)"를 학습.

#### Step 4: Learning Rate로 보수적 업데이트

```
새 예측 = 이전 예측 + learning_rate × 트리 예측
       = 1.2 + 0.05 × 4.2
       = 1.41
```

learning_rate = 0.05이면 트리 보정값의 5%만 반영. 이것이 과적합을 방지하는 핵심 메커니즘.

#### Step 5: 반복 (200\~500회)

잔차 → 트리 → 보정 → 잔차 → 트리 → 보정을 200\~500번 반복. "약과"의 예측이 1.2 → 1.41 → 1.78 → ... → 5.9 로 점진적으로 수렴.

### XGBoost가 표(tabular) 데이터에 강한 이유

1. **트리 분할이 혼합 피처를 자연스럽게 처리**: 연속형(velocity=1.19), 범주형(month=3), 이진(is_holiday=1)을 정규화 없이 동시 입력 가능
2. **비선형 관계와 피처 상호작용을 자동 포착**: "velocity가 높으면서 동시에 positive_ratio도 높을 때만 바이럴" 같은 조합 규칙을 트리가 학습
3. **순차적 잔차 학습**: 이전 트리가 놓친 패턴을 다음 트리가 보완 → 소수 클래스(바이럴)의 잔차가 크므로 자연스럽게 집중 학습

### 클래스 불균형 처리 (분류 변환 시)

**`scale_pos_weight` 파라미터**:
```
scale_pos_weight = 비바이럴 수 / 바이럴 수
예: 바이럴 5% → scale_pos_weight = 95/5 = 19
```
양성 클래스(바이럴) 샘플의 gradient와 hessian에 19배 가중치 적용. 모델이 바이럴을 맞추는 데 19배 더 집중.

회귀 모델에서는 virality_score가 연속값이므로 이 파라미터가 불필요. **이것이 회귀 접근의 장점 중 하나**.

### 설명 가능성: SHAP Values

SHAP(SHapley Additive exPlanations)은 **게임 이론의 Shapley value**에서 유래. 각 피처가 예측에 "얼마나 기여했는지"를 분배.

```
"약과" virality_score 예측 = 5.9

기여 분해:
  기준값 (전체 평균)      = 1.2
  + search_velocity_7d   = +2.3 (가장 큰 기여)
  + positive_ratio       = +1.1
  + mention_velocity_7d  = +0.8
  + ma_ratio             = +0.7
  - competing_trends     = -0.3 (경쟁 키워드 많아서 감소)
  + 기타                  = +0.1
  ───────────────────────
  최종 예측               = 5.9
```

이 분해를 **시각화**하면 "왜 이 키워드가 바이럴로 예측됐는지"를 한눈에 설명 가능. 캡스톤 발표에서 매우 강력한 자료.

### 주요 하이퍼파라미터

| 파라미터 | 역할 | 권장 범위 | 설명 |
|---------|------|----------|------|
| `n_estimators` | 트리 개수 | 100\~500 | 많을수록 정밀하지만 과적합 위험 |
| `max_depth` | 트리 최대 깊이 | 3\~8 | **낮을수록 과적합 방지** (40개 바이럴로는 5\~6 추천) |
| `learning_rate` | 학습률 | 0.01\~0.1 | 낮을수록 보수적, n_estimators와 반비례 관계 |
| `subsample` | 행 샘플링 비율 | 0.7\~0.9 | 각 트리가 데이터의 일부만 사용 → 다양성 |
| `colsample_bytree` | 열 샘플링 비율 | 0.7\~0.9 | 각 트리가 피처의 일부만 사용 |
| `lambda` | L2 정규화 | 1\~5 | 복잡한 트리에 페널티 |
| `alpha` | L1 정규화 | 0\~1 | 불필요한 피처 자동 제거 |

## 4.3 LSTM — 원리와 동작 메커니즘

### LSTM이 해결하는 문제

일반 RNN(Recurrent Neural Network)은 시계열을 순서대로 읽으면서 이전 정보를 전달하는데, **곱셈 기반 전달** 때문에 60일치를 통과하면 초기 정보가 소멸함 (vanishing gradient).

LSTM은 이를 **덧셈 기반 셀 상태(cell state)** 도입으로 해결.

### LSTM 셀 구조: 3개 게이트 + 셀 상태

60일 시계열을 Day 1부터 Day 60까지 순서대로 넣을 때, 각 Day에서 LSTM 셀이 하는 일:

#### 망각 게이트 (Forget Gate)

```
F_t = sigmoid(W_f · [h_{t-1}, x_t] + b_f)
```

- `h_{t-1}`: 어제까지의 요약 (hidden state)
- `x_t`: 오늘의 입력 (search_trend, mention, positive_ratio, ...)
- `W_f`: 학습되는 가중치 행렬
- `sigmoid`: 출력을 0\~1로 압축
- **결과**: 0에 가까우면 "이전 기억 삭제", 1에 가까우면 "이전 기억 유지"
- **예시**: 30일 전의 일시적 스파이크 정보 → F=0.1 (거의 잊기)

#### 입력 게이트 (Input Gate)

```
I_t = sigmoid(W_i · [h_{t-1}, x_t] + b_i)       ← "뭘 저장할지" 결정
C̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c)          ← "저장할 후보 값" 생성
```

- `tanh`: 출력을 -1\~+1로 압축 (양/음 방향 모두 표현)
- `I_t × C̃_t`: 게이트(0\~1)와 후보값(-1\~+1)의 곱 = 실제 저장될 새 정보
- **예시**: 오늘 search_velocity가 2.5로 급등 → I=0.95 (거의 전부 저장)

#### 셀 상태 업데이트

```
C_t = F_t × C_{t-1} + I_t × C̃_t
    = (망각 × 이전 기억) + (입력 × 새 정보)
```

**이 수식이 LSTM의 핵심**. 덧셈(+) 연결이라 60일을 통과해도 gradient가 소멸하지 않음. 일반 RNN은 여기가 곱셈(×)이라 기울기가 기하급수적으로 줄어듦.

#### 출력 게이트 (Output Gate)

```
O_t = sigmoid(W_o · [h_{t-1}, x_t] + b_o)
h_t = O_t × tanh(C_t)
```

- **셀 상태 전체가 아닌, 필요한 부분만 출력**
- Day 60에서 나온 `h_60`이 "이 키워드의 60일간 패턴 요약" 벡터

#### 최종 분류/회귀 레이어

```
h_60 (128차원 벡터)
  ↓
Dense(64, activation=ReLU)
  ↓
Dense(1, activation=None)  ← virality_score 출력 (회귀이므로 활성함수 없음)
```

### LSTM이 본 프로젝트에서 유효한 이유 (회귀 접근)

초기 검토에서 LSTM을 비추했던 근거:
- 이진 분류 시 바이럴 사례 40개 → LSTM이 패턴 외워버림 (과적합)

**회귀로 전환 후 상황이 바뀜:**
- 모든 키워드×모든 시점이 학습 데이터 → 300개 키워드 × 1,400일 = **42만 개 학습 샘플**
- 각 샘플이 고유한 연속 타겟값 보유 → 클래스 불균형 문제 없음
- 42만 개면 LSTM이 의미 있는 시계열 패턴을 학습하기에 충분

### LSTM 구현 시 주의사항 (초보자용)

1. **정규화 필수**: 모든 입력 피처를 0\~1 또는 평균0/표준편차1로 변환
2. **시퀀스 길이**: 60일 권장 (30일은 너무 짧아 계절성 포착 불가, 90일은 학습 부담)
3. **Dropout**: 0.2\~0.3 추천 (과적합 방지)
4. **Learning Rate**: 0.001 (Adam optimizer 기본값)에서 시작
5. **Early Stopping**: 검증 loss가 10 epoch 연속 개선 안 되면 중단

## 4.4 베이스라인 모델

캡스톤에서 "메인 모델이 얼마나 나은지" 비교할 기준이 필수. 회귀 태스크이므로 회귀 베이스라인 2개를 사용한다.

### 베이스라인 ①: Dummy Model (항상 평균 예측)

```python
from sklearn.dummy import DummyRegressor
dummy = DummyRegressor(strategy='mean')
dummy.fit(X_train, y_train)
# 모든 샘플에 대해 y_train의 평균값(예: 1.2)을 예측
```

피처를 전혀 보지 않고 학습 데이터의 평균만 예측한다. 이 모델을 이기지 못하면 "피처가 아무 정보도 담고 있지 않다"는 뜻이므로, 모델이 의미 있는지 판별하는 최소 기준선이다.

예를 들어 Dummy RMSE = 3.5인데 XGBoost RMSE = 1.8이면, XGBoost가 "평균만 찍는 것 대비 49% 개선"이라고 보고할 수 있다.

### 베이스라인 ②: Linear Regression (선형 결합)

```python
from sklearn.linear_model import Ridge
lr = Ridge(alpha=1.0)
lr.fit(X_train, y_train)
# pred = w₁×velocity + w₂×ma_ratio + ... + w₃₈×csi + b
```

43개 피처의 선형 결합으로 예측. Ridge(L2 정규화)를 사용하여 다중공선성에 안정적으로 대응. 각 피처의 계수 `w_i`가 직접 해석 가능:

```
w_velocity = 0.85 → velocity가 1 증가하면 virality_score가 0.85 증가
w_positive_ratio = 1.2 → 긍정 비율이 1 증가하면 score가 1.2 증가
```

Dummy와 Linear Regression 사이의 성능 차이는 "피처의 선형적 예측력"을 보여주고, Linear Regression과 XGBoost 사이의 차이는 "비선형 모델링의 가치"를 보여준다.

### 한계

- **선형 결합만 가능**: "velocity가 높으면서 동시에 sentiment도 높을 때만 바이럴" 같은 피처 상호작용을 학습 못함
- **비선형 관계 무시**: 실제로는 velocity가 어느 임계점을 넘어야 바이럴이 급등하는데, 선형 회귀는 이를 모델링 불가

### 발표용 스토리 구조

```
Dummy RMSE = 3.5  →  "아무것도 안 하면 이 정도 틀림"
  ↓ (피처의 선형적 가치)
Linear Reg RMSE = 2.4  →  "피처가 유의미하긴 하나 선형으로는 한계"
  ↓ (비선형 모델링의 가치)
XGBoost RMSE = 1.8  →  "트리 기반 비선형 학습으로 추가 개선"
  ↓ (시계열 패턴의 가치)
LSTM RMSE = 1.9  →  "시계열 패턴은 약간의 추가 정보 제공"
  ↓ (앙상블의 가치)
Ensemble RMSE = 1.6  →  "서로 다른 입력을 보는 모델의 조합이 최종 개선"
```

위 수치는 예시이며 실제 실험에서 달라질 수 있다. 순서가 뒤집히는 것도 의미 있는 분석 포인트.

## 4.5 검토 후 제외한 모델들

### Transformer — 원리 이해 후 제외 결정

**Self-Attention 메커니즘 원리:**

LSTM이 Day 1 → Day 2 → ... → Day 60을 순서대로 읽는 반면, Transformer는 모든 날짜를 동시에 보면서 "어떤 날짜가 어떤 날짜와 관련 있는지"를 학습.

각 시점(Day)이 세 가지 벡터를 생성:
- **Query (Q)**: "나는 어떤 정보가 필요해?" — 현재 시점의 질의
- **Key (K)**: "나는 이런 정보를 갖고 있어" — 모든 시점의 식별자
- **Value (V)**: "내 실제 데이터는 이거야" — 모든 시점의 실제 값

```
Attention(Q, K, V) = softmax(Q · K^T / √d) × V
```

Q와 K의 내적 → 유사도 계산 → softmax로 가중치 변환 → V에 적용. 즉 "Day 55의 급등이 Day 30의 뉴스 폭증과 관련 있다"는 걸 직접 연결할 수 있음. Multi-head attention은 이를 여러 "관점"에서 동시에 수행.

**제외 사유:**
- 데이터 요구량이 LSTM보다 높음 (50K+ 샘플 권장)
- 하이퍼파라미터가 매우 많음 (attention head 수, dimension, block 수, FF 차원 등)
- GPU 필수에 가까움
- 바이럴 사례 40\~80개에서는 과적합 거의 확실
- 팀 ML 경험 없음 → 구현·디버깅 난이도 과도
- **결론: 원리는 이해하되 이 프로젝트에서는 사용하지 않음**

### Facebook Prophet — 분류/회귀 불가, 피처 생성기로 활용 가능

- 시계열을 trend + seasonality + holidays로 분해하는 모델
- "검색량이 내일 얼마일까"를 예측하는 **시계열 예측(forecasting)** 전용
- "바이럴 될까?" 같은 분류/회귀 문제에 직접 사용 불가

**피처 생성기(Feature Generator)로 활용하는 하이브리드 접근:**

각 키워드의 과거 시계열에 Prophet을 적합(fit)하고, 추출한 컴포넌트를 XGBoost의 추가 피처로 사용:

```
키워드 "약과"에 Prophet 적합 → 추출 피처:
  - trend_slope: 전반적 상승/하락 방향 (+0.12)
  - seasonality_amplitude: 계절 변동 폭 (15.3)
  - forecast_residual: 실제값 - Prophet 예측값 (큰 양수 = Prophet도 예측 못한 급등)
```

이 피처들을 기존 피처에 추가하면 XGBoost에 +3\~7% AUC 개선 기대. 시간 여유 시 고려.

### ARIMA/SARIMA — 제약이 많으나 피처 생성기로는 가능

- Autoregressive 모델: 과거 값의 선형 조합으로 미래 값 예측
- **정상성(stationarity) 가정 필요**: 평균·분산이 시간에 따라 변하면 안 됨 → 식품 트렌드는 비정상적
- 분류 불가, 회귀도 단변량(검색량 하나)만 가능

**피처 생성기로 활용:** Prophet과 마찬가지로 ARIMA 적합 후 AR 계수, 잔차 분산 등을 추출하여 XGBoost 피처로 사용 가능. 다만 Prophet보다 구현이 복잡하고 이점이 적어 우선순위 낮음.

### Random Forest — 선택적 추가

- XGBoost와 동일한 트리 기반이지만 **독립적으로** 트리를 만들고 다수결
- 과적합에 강하나, 소수 클래스 학습이 XGBoost보다 약함
- 시간 여유 시 추가 비교 대상으로 권장 (XGBoost와 트리 기반 내 성능 차이 분석용)

| **설명 가능성** | N/A | 계수 직접 해석 | **SHAP (개별 예측 설명)** | 해석 매우 어려움 |

### 학술 근거

**Grinsztajn et al. (2022) — "Why do tree-based models still outperform deep learning on tabular data?"**

- 45개 데이터셋에서 XGBoost, Random Forest vs 딥러닝 비교
- **결과**: 표 형태 데이터에서 트리 모델이 80% 이상의 경우 딥러닝을 이기거나 동등
- 특히 10만 샘플 이하에서 트리 모델이 결정적 우위
- **본 프로젝트 시사점**: XGBoost가 LSTM보다 나을 가능성 높으나, 회귀 접근(42만 샘플)에서는 격차가 줄어들 것

**캡스톤 발표 시 활용**: "Grinsztajn 논문의 결론과 일치하게, 본 프로젝트에서도 XGBoost가 LSTM 대비 소폭 우세했습니다" 또는 "회귀 접근 덕분에 LSTM이 충분한 데이터를 확보하여 유사한 성능을 달성했습니다" — 어느 결과가 나오든 학술적으로 해석 가능.

**왜 이렇게 가벼운가:**

- **XGBoost**: `tree_method='hist'`가 피처 연속값을 256개 bin으로 이산화하여 처리. 25개 피처 × 40만 행도 bin으로 압축하면 실질 계산량이 매우 작음. GPU 없이도 수 분 내 완료.
- **LSTM**: 매 에폭마다 40만 × 60 timestep × 4개 게이트 행렬곱 수행. GPU 병렬화 효과가 극대화되는 구조(T4 CUDA 코어 2560개 동시 연산)로 CPU 대비 5\~10배 빨라짐.
- **피처 수 증가 영향**: Group E(연관어 TF-IDF+PCA 20차원) + Group F(경제 2개) 추가로 XGBoost 피처가 16→43개로 늘어났지만, histogram binning 덕분에 학습 시간 차이는 체감 불가 수준(10\~15% 증가).

## 4.8 XGBoost + LSTM 앙상블

### 왜 이 조합인가

앙상블의 핵심 원리는 **다양성(diversity)** — 서로 다른 방식으로 틀리는 모델을 합치면 개별 모델보다 안정적인 예측이 나온다. XGBoost와 LSTM은 입력 자체가 다르기 때문에 다양성이 높다:

| | XGBoost | LSTM |
|---|---|---|
| 입력 | t시점 파생 피처 43개 (스냅샷) | 60일 raw 시계열 10\~15채널 |
| 강점 | 피처 간 비선형 상호작용 포착 | 시간 순서 패턴 자동 추출 |
| 약점 | 시간 흐름을 직접 보지 못함 | 수치 피처 간 복합 조건 약함 |

예를 들어 "두바이 초콜릿"의 경우, XGBoost는 "velocity_7d = 1.8, positive_ratio = 0.75 → 바이럴"이라고 판단하고, LSTM은 "60일 시계열에서 최근 10일간 점진적 상승 곡선 → 바이럴"이라고 판단한다. 둘이 같은 결론에 도달하더라도 **근거가 다르기 때문에**, 한쪽이 틀렸을 때 다른 쪽이 보정해줄 가능성이 높다.

반면 동일한 43개 피처를 입력받는 모델끼리(예: XGBoost + Random Forest + Linear Regression) 앙상블하면, 같은 데이터를 보고 비슷한 실수를 하므로 다양성이 낮아 개선 효과가 제한적이다.

### 조합 방식: Weighted Average (가중 평균)

```python
# validation set에서 최적 가중치 탐색
best_alpha, best_rmse = 0, float('inf')
for alpha in [0.0, 0.1, 0.2, ..., 1.0]:
    pred_ensemble = alpha * pred_xgb + (1 - alpha) * pred_lstm
    rmse = compute_rmse(pred_ensemble, y_val)
    if rmse < best_rmse:
        best_alpha, best_rmse = alpha, rmse

# 테스트 시 적용
final_pred = best_alpha * pred_xgb_test + (1 - best_alpha) * pred_lstm_test
```

α = 1.0이면 XGBoost만, α = 0.0이면 LSTM만 사용하는 것과 동일. 앙상블이 개별 모델보다 무조건 같거나 나은 성능을 보장한다 (validation 기준으로 α를 선택하므로).

### 다른 조합 방식과의 비교

| 방식 | 구현 난이도 | 성능 | 비고 |
|---|---|---|---|
| **Simple Average** (α=0.5 고정) | 코드 1줄 | 보통 | 가중치 최적화 없음 |
| **Weighted Average** | 코드 5줄 | 좋음 | **본 프로젝트 채택** |
| **Stacking** (meta-model 학습) | 코드 30줄+ | 약간 더 좋음 | meta-model용 별도 validation fold 필요, 시계열 분할과 엮이면 복잡 |

Stacking은 meta-model(예: Ridge Regression)이 "두 모델의 예측값"을 입력으로 받아 최종 예측을 산출하는 방식. Weighted Average보다 유연하지만, 시계열 데이터에서 meta-model 학습용 fold를 별도로 설계해야 하고 데이터 누수 위험이 생긴다. 성능 차이는 통상 0.5\~1% 수준으로, 학부 캡스톤에서는 복잡성 대비 이득이 적다.

### 추가 학습 시간

앙상블 자체에는 추가 학습이 없다 — XGBoost와 LSTM을 각각 학습한 후, validation set 예측값으로 α를 탐색하는 것뿐. α 탐색은 11개 값(0.0\~1.0, 0.1 간격) × RMSE 계산이므로 1초 미만.

### 주의: SHAP 해석

앙상블 모델의 SHAP 값은 직접 계산할 수 없다. 대신 개별 모델의 SHAP을 각각 보고하면 된다:

- XGBoost SHAP: "어떤 피처가 예측에 기여했는가" (43개 피처 기여도)
- LSTM: Attention weight 또는 gradient-based attribution으로 "어느 시점이 예측에 기여했는가"

발표 시에는 XGBoost SHAP을 메인으로 보여주고, "LSTM은 시계열 패턴을 보완하여 앙상블 RMSE를 N% 개선"이라고 정리하면 충분하다.
